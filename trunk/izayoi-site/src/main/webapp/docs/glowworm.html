our goal of design pattern

Glowworm is a tiny tiny name mapping based dependency injector for web frontend development.

  1. Glowworm 是一个功能非常、非常有限的依赖注入工具。Glowworm 仅能够在每次请求中根据请求地址，从相似名称的数据脚本文件创建对象；也仅能够将创建的对象注入到请求上下文。相对的，这一基于名称映射的约定不需要配置或后台编码，这使得 web 前端的开发人员可以随时简单地为页面提供测试用的渲染数据，而不必等待与之相关的后台系统环境部署完成。

  2. Glowworm 默认支持的数据描述格式包括纯文本、JSON、properties 文件、JSP 以及 MVEL2 脚本。用户也可以向 Glowworm 挂载自己编制的注入器，并映射到某个特定的扩展名。

  3. Glowworm 的主要设计目标是方便前端开发者观察页面的渲染效果。利用 Glowworm 为前端提供来自业务逻辑的真实数据可能会造成一定程度的效率问题——虽然在一个足够简单的系统中你确实可以这么做。
