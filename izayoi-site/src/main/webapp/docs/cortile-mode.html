cortile landscape

Scenery Mode

这是 Cortile 的标准模式。将 CortileScenery 类绑定到 *.html 上，用户访问 index.html，Cortile 即将 index.html 作为页面模板编译。

根据与 JSP 相似的风格，CortileScenery 类通常被配置为 Servlet；但它也可以配置为 Filter 并实现完全相同的功能。 如果你在使用一个支持 Filter 链的 MVC 框架（例如 Spring），Filter 类型可能会比 Servlet 类型更容易利用一些。

Mirage Mode

Mirage 模式被设计为用来为非 html 类型的文件编写模板。这一模式通过基于一个中缀名（默认为 .mirage）的名称映射发挥作用：
例如同目录下存在 conf.properties 以及 conf.mirage.properties，当用户访问 conf.properties 时，CortileMirage 拦截请求，将 conf.mirage.properties 作为模板编译，并将请求转发过去。

这一设计产生重复代码，但它是原始文件在脱离服务器的静态环境中正常调试的必要保证。虽然直接将 CortileScenery 关联到 .properties 文件也可以进行模板的编制，但由于用于 html 的注释语法无法在 .properties 文件中作为注释被忽略；其结果就是当一段 js 代码试图在静态环境下读取 properties 文件的时候，额外的 html 注释就会造成程序运行的失败。

Mirage Mode 默认使用 ELTextCompiler 编译模板；要换用其它的编译器，需要在 .mirage 后添加 -类型 的后缀；例如，如果希望将 conf.properties 的模板使用默认的 HTMLCompiler 进行编译，那么它的模板文件名应为 conf.mirage-html.properties。

CortileMirage 类只能被配置为 Filter。将所有 html 文件也利用 CortileMirage 处理也是合法的，但由于重复代码的问题，不推荐这样使用。
