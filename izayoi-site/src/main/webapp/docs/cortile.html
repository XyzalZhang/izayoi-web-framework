our goal of design pattern

Cortile is a standalone, totally HTML compatible dynamic page engine.

  1. Cortile 是单纯的视图层技术。在未经自行扩展的前提下，由 Cortile 所构建的页面只响应直接发往该页面的请求，并只从请求上下文中取得数据。除最低程度的必要转发外 Cortile 无法参与请求/响应、业务逻辑及数据交换等后台机制的管理；但另一方面，这种机制也使得任何一个 web 开发框架只要能够结合传统的 JSP 页，就一定可以通过同样的方式结合 Cortile。

  2. Cortile 的语法完全基于 HTML 注释与标签属性；这种机制回避了因引入额外标签或非 XML 代码造成的文档 DOM 结构的变化，也就是说，通过 Cortile 构建的动态页面可以脱离 J2EE 容器，作为静态 HTML 页进行正常的 HTML、CSS、Javascript 编程调试。

  3. Cortile 直接提供包括 EL 表达式、页面片段递归等在内的一系列语法；对于更加复杂的需求，Cortile 允许用户向编译器挂载自己编制的语法类、EL 解释器（系统默认使用 MVEL2 作为 EL　脚本的引擎），或者直接向 Cortile 引擎添加新的编译器实现。

  4. Cortile 在第一次访问时将页面编译成 JSP，之后向其转发所有发往原页面的请求；这一机制使得 Cortile 能够在仅有 JRE 的场合利用 Servlet 容器获得尽可能高的运行效率，也使得 Cortile 得以提供一些辅助语法，以允许用户继续利用传统的 JSP 代码（这并不推荐，但它可以帮助用户顺利过渡）。
