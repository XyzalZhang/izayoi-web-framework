cortile html compatible
dynamic page engine

Cortile ONE-MINUTE QUICK START

  • Step 1: web.xml

  • Step 2: index.html

  • Done. browse http://yourhost/index.html

Cortile is a standalone, totally HTML compatible dynamic page engine.

  1. Cortile 的语法完全基于 HTML 注释与标签属性；这种机制回避了因引入额外标签或非 XML 代码造成的文档 DOM 结构的变化，也就是说，通过 Cortile 构建的动态页面可以脱离 J2EE 容器，作为静态 HTML 页进行正常的 HTML、CSS、Javascript 编程调试。

  2. Cortile 是单纯的视图层技术。在未经自行扩展的前提下，由 Cortile 所构建的页面只响应直接发往该页面的请求，并只从请求上下文中取得数据。除最低程度的必要转发外 Cortile 无法参与请求/响应、业务逻辑及数据交换等后台机制的管理；但另一方面，这种机制也使得任何一个 web 开发框架只要能够结合传统的 JSP 页，就一定可以通过同样的方式结合 Cortile。

  3. Cortile 直接提供包括 EL 表达式、页面片段递归等在内的一系列语法；对于更加复杂的需求，Cortile 允许用户向编译器挂载自己编制的语法类、EL 解释器（系统默认使用 MVEL2 作为 EL　脚本的引擎），或者直接向 Cortile 引擎添加新的编译器实现。

  4. Cortile 在第一次访问时将页面编译成 JSP，之后向其转发所有发往原页面的请求；这一机制使得 Cortile 能够在仅有 JRE 的场合利用 Servlet 容器获得尽可能高的运行效率，也使得 Cortile 得以提供一些辅助语法，以允许用户继续利用传统的 JSP 代码（这并不推荐，但它可以帮助用户顺利过渡）。

glowworm tiny dependency injector
for web frontend development

Glowworm ONE-MINUTE QUICK START

  • Step 1: web.xml

  • Step 2: index.jsp

  • Step 3: index.jsp.gw.json

  • Done. browse http://yourhost/index.jsp

Glowworm is a tiny tiny name mapping based dependency injector for web frontend development.

  1. Glowworm 是一个功能非常、非常有限的依赖注入工具。Glowworm 仅能够在每次请求中根据请求地址，从相似名称的数据脚本文件创建对象；也仅能够将创建的对象注入到请求上下文。相对的，这一基于名称映射的约定不需要任何配置文件或后台代码，这使得在一个分层系统的开发过程中，web 前端的开发人员可以在不连接后台系统的情况下，简单地为前端页面提供渲染所需的基本测试数据。

  2. Glowworm 默认支持的数据描述格式包括纯文本、JSON、properties 文件、JSP 以及 MVEL2 脚本。用户也可以向 Glowworm 挂载自己编制的注入器，并映射到某个特定的扩展名。

  3. Glowworm 的主要设计目标是方便前端开发者观察页面的渲染效果。利用 Glowworm 为前端提供来自业务逻辑的真实数据可能会造成一定程度的效率问题——虽然在一个足够简单的系统中你确实可以这么做。
